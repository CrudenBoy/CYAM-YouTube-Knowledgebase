# CYAM Apps Script Integration Handover

## 1. Project Context
The **Multimodal Video Indexing Pipeline** (built in Python) has been completed and packaged into a standalone Mac desktop application located in `CYAM-YouTube-Knowledgebase`. It takes a `batch_manifest.csv` and outputs a highly structured, RAG-ready `final_video_index.csv` containing hierarchical JSON steps, explicit metadata, and timestamped video links.

We are now transitioning to the **CYAM-Workspace** to update the Google Apps Script (`CYAM_Master_Knowledge_Index` project) to support this new "Sandwich Workflow" data structure.

## 2. Immediate Objectives for the AI

### Task 1: Legacy Code Cleanup
*   **Remove Obsolete Menus**: In the `onOpen` function, delete the menu items for **"Add YouTube Video Wizard"** and **"Slice Transcripts by Chapter"**.
*   **Delete HTML Assets**: Remove the `.html` files associated with these two obsolete features.

### Task 2: Build the "Batch Link Ingestion" UI (The Queue)
*   **Context**: The user needs a simple way to queue up new videos for the local Mac pipeline.
*   **Action**: Create a new modal dialogue (e.g., "Batch Add Video Links") in Apps Script.
*   **Requirements**:
    *   A large text area allowing the user to paste multiple video links.
    *   A single category dropdown selector mapping to `dkd_doc_ref` tags (Options: Opal, Google Workspace Studio, Apps Script, N8N).
    *   An execution script that appends these new URLs to the bottom of the `YT_Video_Index` spreadsheet tab, filling the `dkd_doc_ref` column.
*   **UX Note**: The user will then naturally download this Google Sheet as a CSV, run it through the Mac `Launch_Indexer.command` tool, and paste the enriched output back over the blank rows.

### Task 3: Update JSON Exporter Schema (The Output)
*   **Context**: The `Export YouTube Knowledge JSON` function currently ignores the new Python columns because it is hardcoded to a legacy schema.
*   **Action**: Update the mapping logic in the exporter to include the full 16-column structure produced by the pipeline so it exports properly to the CDN.
*   **Critical Columns to Add**:
    *   `url_with_timestamp` (String)
    *   `detailed_steps` (Stringified JSON - ensure it passes through the exporter correctly)
    *   `faq_question` / `faq_answer`
    *   `difficulty_level` / `prerequisites`
*   *Note: Remind the user that if the script says "Success" but the file isn't updating on the CDN, it's likely because the script is successfully overwriting a deleted version of the file inside the Google Drive Trash via a hardcoded File ID.*

## 3. Data Contract Reference
The exact headers generated by the Mac tool (and expected by the `YT_Video_Index` sheet) are:
`video_url, video_id, video_title, platform, publish_date, chapter_title, timestamp_seconds, url_with_timestamp, transcript_chunk, dkd_doc_ref, key_concepts, faq_question, faq_answer, difficulty_level, prerequisites, detailed_steps`
